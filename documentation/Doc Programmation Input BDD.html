Partie Tableur To BDD

Ce document va expliquer le fonctionnement du code pour stocker les valeurs dans les différents tableurs vers la bdd

Les tableurs d’entrées

Les tableurs contiennent les infos rentrées par l’utilisateur, attention au type des infos attendues: bool, int , float etc sinon les scripts python vont renvoyer des erreurs

InputMeshs doit avoir les meshs regroupé par scène et espacé d’une entrée vide pour permettre la recherche par id une fois dans la bdd

InputScene relie le 1er mesh d’une scène: le 1er dans le tableur d’input des meshs, au nom de la scène

InputSeq chaque séquenceur peut être relié à un ou plusieurs sons, le 1er son relié va être joué en 1er et etc

InputSon, attention le temps est en seconde, pas de virgule. Volume et Spatial sont des booleans .

Attention les tableurs doivent être au format .csv pour le bon fonctionnement des scripts python

Le code python

Le code et commenté, le principe est le suivant :
On parse les fichiers csv correspondant et on les stocks dans la bdd.
Certaines opérations permettent de récupérer l’id de l’objet suivant et de check si il y en a un (ou de passer à la scène suivante)
